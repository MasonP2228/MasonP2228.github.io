Robotic Reef

As a follow-up to the robotic light drawing group project, I was tasked with making a second light drawing individually. For this second project, I used the same approach of mounting a microcontroller on the university's KUKA robotic arm and taking a long exposure photograph of its path of motion. This time, however, I wanted to create an actual picture instead of just an abstract pattern.
I settled on a design of a sea turtle swimming through a coral reef. I chose this design because I felt that it would lend itself well to the generative algorithms we had been learning about in class. To make the final image seen here, I actually made 4 different light drawings, and I used Photoshop to combine them together. Each light drawing was made using the Grasshopper parametric design software, and 3 of the scripts integrated Python code as well.
The most complicated light drawings were the two pieces of coral. To create these, I had to implement a stack data structure to create a backtracking logic. This was necessary because the robot's movement commands travel from point to point sequentially. However, coral has a branching structure, so the robot was drawing lines directly from the end of one branch to the base of another. Integrating Python code within my Grasshopper scripts helped me to solve issues like this.
